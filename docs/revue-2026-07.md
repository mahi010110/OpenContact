# OpenContact — revue générale (juillet 2026)

Revue complète demandée par le mainteneur : sécurité d'abord, puis
performances, qualité, fiabilité. Périmètre : tout le moteur (`engine/`),
tous les écrans (`ui/`), `app.js`, `sw.js`, `index.html`, `oauth.html`,
le Compagnon (`compagnon/coeur`, `compagnon/src-tauri`), les tests et la CI —
à la lumière des audits précédents (`docs/audit-securite.md`,
`docs/fable5/ANALYSE-SECU-PERF.md`), dont les constats ont été re-vérifiés
dans le code actuel.

**Bilan honnête : le socle est sain.** Les surfaces classiques (XSS,
injection, bombes de décompression, pollution de prototype, fuites
cross-origin) sont couvertes et testées ; la crypto (OC2, coffre OCV1,
anneau Ed25519, canal du Compagnon) est conforme à l'état de l'art.
Cette passe corrige **quatre points réels** et deux finitions — rien
de plus n'a mérité d'être touché.

## Corrigé

### R1 — Canal local du Compagnon : corps HTTP borné · sécurité · `canal.rs`

Le serveur 127.0.0.1 répond volontairement à toute origine web (la
sécurité est dans le chiffre, pas dans l'origine — S2 de l'audit), mais
`read_to_string` chargeait le corps **entier** en mémoire, sans limite :
n'importe quelle page web visitée pouvait poster des gigaoctets et
épuiser la mémoire du Compagnon. **Corrigé** : lecture bornée à 8 Mo
(le plus gros corps légitime est une mission-campagne scellée — données
plafonnées à 4 Mo côté PWA, plus l'enveloppe base64) ; au-delà, refus
`413 troplourd`. Le surplus est vidé **par petits blocs** avant la
réponse : le nettoyage interne de tiny_http (`EqualReader::drop`)
allouerait sinon d'un seul coup toute la taille annoncée restante —
le chemin de rejet ne doit pas rouvrir la porte qu'il ferme.

### R2 — La re-preuve du code partage le garde-fou d'essais · sécurité · `ui/verrou.js`

L'écran verrouillé impose un délai progressif après 5 échecs
(compteur persistant `meta.guard`), mais les feuilles de re-preuve
(`requireCode` : restaurer, connexions, rompre le lien, campagnes…)
offraient des **essais illimités** — incohérent avec le modèle
« téléphone déverrouillé emprunté » qui justifie la re-preuve.
**Corrigé** : un pavé commun (`proofPad`) applique le **même compteur**
que l'écran verrouillé — 5 échecs n'importe où = délai partout, le bon
code efface tout, comme avant. Au passage, ces feuilles acceptent
maintenant la **saisie clavier** (ordinateur), comme l'écran verrouillé.
Couvert par la section 5bis d'`e2e-verrou.mjs` : 5 échecs au clavier →
délai affiché, retrouvé à l'écran verrouillé après rechargement, levé à
l'expiration.

### R3 — Date d'exemple décalée la nuit · fiabilité · `ui/state.js`

`addDemo` calculait la date de prochaine action avec `toISOString()`
(UTC) — le piège exact que `engine/utils.js` documente : entre minuit
et 2 h (heure de France), la piste d'exemple « aujourd'hui » tombait la
veille et s'affichait en retard. **Corrigé** : `localISO()`, comme
partout ailleurs. (Les deux usages restants de `toISOString` dans le
moteur sont **volontaires** : migration v5 épinglée par un vecteur de
test UTC, et résumé MCP miroité par des fixtures croisées Rust.)

### R4 — Toast doublement échappé · qualité · `ui/pistes.js`

Le glisser-déposer d'un statut passait le nom par `esc()` avant
`toast()`, qui affiche du texte brut (`createTextNode`) : « R&D »
devenait « R&amp;D ». **Corrigé** : texte brut.

## Finitions

- **Démarrage** (`app.js`) : les quatre chargements différés
  (messagerie, campagnes, analyse, propositions) étaient attendus en
  série ; ils sont indépendants et partent maintenant en parallèle —
  quatre allers-retours IndexedDB de moins dans le chemin critique.
- **`sw.js`** : bump `oc-v35` (imposé par les fichiers précachés
  modifiés) ; la branche morte `r.type === 'opaque'` — inatteignable
  puisque seul le même-origine passe — est retirée avec son commentaire
  trompeur (« libs CDN » : il n'y en a pas).

## Constaté, volontairement non touché

Compromis documentés et cohérents, re-vérifiés mais pas re-tranchés :

- **PIN 6 chiffres** (D6) : le secret fort reste la phrase (~96 bits) ;
- **géocodage pendant la frappe** (`engine/geo.js`) : arbitrage
  explicitement laissé au mainteneur (audit §« Arbitrages », point 1) ;
- **CORS `*` + Private-Network** du canal local (S2) et absence
  d'anti-rejeu de séquence (S3) : modèle de menace assumé ;
- **micro-optimisations de rendu** (`campaignOfPiste` par ligne,
  `blobOf` par frappe) : bornées par les caps d'affichage, gain < risque ;
- **`VIEW_KEY`** : pierre tombale de contrat, protège le namespace de clés.
